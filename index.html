HTML

  • The head element contains information about the webpage.
  • The body element represents the visiblecontent shown to the user.
  • Hypertext Markup Language, gives the content structure.
  • Metadata is data about our webpage that the browser usesto affectively serve our webpage to users. It is not typically seen by users, but it contains important information about accesibility, search engines, and performance.
  • HTML Attribute, defines additional information about an element and is always defined in the beginning element, never the closing element.
  • SEO, the process of improving the ranking of a website in search engines, such as Google.
  • Script, embeds client-side scripts into the HTML doc.
  • Style, defines the CSS styles associated with a website.
  • Base, defines the base URL for a website.
  • Link, connects an external resource to the HTML doc.
  • Meta, defines metadata such as character set (UTF-8), description, keyword, author, and viewport.
  • HTTP-equiv, sets an HTTP header for content.
  • Charset, sets our character encoding.
  • Viewport, sets the default view for our wqebpage.
  • Child Elements, elements nested inside other elements.
  • Parent Elements, The element that holds all of the Child Elements.
  • Semantic HTML Elements, the name of the element describes its content and the role it plays on a webpage.
  • Div, creates a division in a page.
  • Span, creates an inline division.
  • Element, refers to the entire element from opening tag to closing tag.
  • Tag, refers to only what appears inside the angle brackets.
  • H1-6, represents the level of heading a given text block represents.
  • P, represents a paragraph or block of text.
  • Ul, unordered lists.
  • Ol, unordered lists.
  • Li, line item.
  • Img, contains information abpout images that are displayed on the webpage.
  • Src, attribute that defines the location of an image.
  • Br, creates a line of empty space, or a line break, between two breaks of contyent.
  • A, anchor, creates links to the same webpage or other webapges.
  • Href, points to the URL for the link.
  • Alt, contains a text stringthat describes the appearance and funtionality of an image.
  • Section, semantic elements that define a section in a documentthat conatains thematic content.
  • Id, a way to identify elements and select them.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicateshow much space we want around the content inside an element.
  • Cascading Style Sheets- programming language used by developers to define how webpage content is presented to users. CSS defines the colors, aesthetics, visual layout and organization. We use CSS to define style rules for specific elements of an HTML page and format them using a wide range of style options.
  • Inline CSS- Apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only.
  • Internal Style Sheet- Contains CSS rules for the webpage in the head section of the HTML file.
  • External Style Sheet- allows developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • Separation of Concerns- Separating code into different files based on use.
  • Declarations- Contain two important components; the CSS property we want to apply and the value of the property.
  • DRY- Do Not Repeat Yourself- A coding principal that encourages developers to reuse or share pieces of code in order to reduce the number of lines of code the need to be written, or maintained.
  • Assigning colors in CSS can be done in many ways, including by semantic references such as gray or blue. We can also refer to a color by its hexidecimal code or hex#.

Git

  • Git Status, checks what branch we are currently on.
  • Git Checkout -b branch-name, creates a new branch and switches to it.
  • Git Add, adds a file to yur next commit as it looks now.
  • Git Reset, unstage a file while retaining the changes in the working directory.
  • Git Diff, diff of what is changed but not staged.
  • Git Diff --Staged, diff of what is staged but not yet commited.
  • Git Commit -m "descriptive message", commit your new staged content as a new commit snapshot.
  • Git Branch, lists your branches, .a will appear next to the currently active branch.
  • Git Branch [branch name], create a new branch at the current commit.
  • Git Merge, merge the specified branches history into the current one.
  • Git Log, show all commits in the current branches history.
  • Git Config --Global User.Name "[firstname lastname]", set a name that is identifiable for credit when review revision history.
  • Git Config --Global User.Email "[valid email]", set an email address that will be associated with each history maker.
  • Git Config --Global Color.ui Auto, set automatic line coloring for git for easy reviewing.
  • Git Init, initialize an existing directory as a Git repository.
  • Git Clone [URL], retrieve an entire repository from a hosted location via URL
  • Git Log BranchB..BranchA, show the commits on Branch A that are not on Branch B.
  • Git Log --Follow [file], show the commits that changed file, even accross renames.
  • Git Diff BranchB...BrnachA, show the difference of what is i branch A that is not in Branch B.
  • Git Show [SHA], show any object in Git in human readable format.
  • Git RM, delete the file from project and stage the removal for commit.
  • Git MV [existing-path] [new-path], change an existing file path and stage the move.
  • Git Log --stat -M, show all commit logs with indication of any paths that moved.
  • Git Config --global core.excludesfile [file], system wide ignore pattern for all local repositories.
  • Git Remote Add [Alias] [URL], add a git URL as an alias.
  • Git Fetch [Alias], fetch down all the branches from that Git remote.
  • Git Merge [Alias]/[Branch], merge a remote branch into your current branch to bring it up to date.
  • Git Push [Alias][Branch], transmit local branch commits to the remote repository branch.
  • Git Pull, fetch and merge any commits from the tracking remote branch.
  • Git Rebase [Branch], apply any commits of current branch ahead of specified one.
  • Git Reset --Hard [Commit], clear staging area, rewrite working tree from special commit.
  • Git Stash, save modified and staged commands.
  • Git Stash List, list stack-order of stashed file changes.
  • Git Stash Pop, write working from top stash stack.
  • Git Stash Drop, discard the changes from the top stash stack.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • JavaScript, a programming language the adds interactivity to your website.
  • API, Browser Apllication Programming Interfaces, built into web browsers, providing functionality such as dynamically creating HTML and setting CSS styles.
  • Third Party API, allow developers to incorporate functionality in sites from other content providers, such as Twitter, or Facebook.
  • Third Party Frameworks, and Libraries, can be applied to HTML to accelerate the work of building sites and apps.
  • Var, keyword used to declare a variable.
  • =, assigns specific data we want stored in our variable.
  • Each line of JavaScript code ends with a semicolon.
  • Console.log, output a message to the web console by creating an argument.
  • Control Flow, the order in which a computer executes code in a script. In JavaScript the code runs from the first statement at the top of the page, to the last statement at the bottom.
  • If Statement, a way of introducing desicion-making into our code. Code will only execute if the statement meets a condition or is true.
  • Else If, used if there is more than one condition.
  • The Strict Equality Operator, "===", checks to see if two values are equal, and returns a Boolean result "true" if the values are equal "false" if they are not equal.
  • Array, a single variable that is used to hold a group of data.
  • For Loop, uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
  • Commenting Out Code, makes it inactive without deleting it, just in case you need it later.
  • OOP, Object Oriented Programming.
  • Function, a function is different from a loop or conditional statement because functions do not automatically execute when the JavaScript file is run.
