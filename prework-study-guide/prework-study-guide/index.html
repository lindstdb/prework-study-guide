HTML

  • The head element contains information about the webpage.
  • The body element represents the visiblecontent shown to the user.
  • Hypertext Markup Language, gives the content structure.
  • Metadata is data about our webpage that the browser usesto affectively serve our webpage to users. It is not typically seen by users, but it contains important information about accesibility, search engines, and performance.
  • HTML Attribute, defines additional information about an element and is always defined in the beginning element, never the closing element.
  • SEO, the process of improving the ranking of a website in search engines, such as Google.
  • Script, embeds client-side scripts into the HTML doc.
  • Style, defines the CSS styles associated with a website.
  • Base, defines the base URL for a website.
  • Link, connects an external resource to the HTML doc.
  • Meta, defines metadata such as character set (UTF-8), description, keyword, author, and viewport.
  • HTTP-equiv, sets an HTTP header for content.
  • Charset, sets our character encoding.
  • Viewport, sets the default view for our wqebpage.
  • Child Elements, elements nested inside other elements.
  • Parent Elements, The element that holds all of the Child Elements.
  • Semantic HTML Elements, the name of the element describes its content and the role it plays on a webpage.
  • Div, creates a division in a page.
  • Span, creates an inline division.
  • Element, refers to the entire element from opening tag to closing tag.
  • Tag, refers to only what appears inside the angle brackets.
  • H1-6, represents the level of heading a given text block represents.
  • P, represents a paragraph or block of text.
  • Ul, unordered lists.
  • Ol, unordered lists.
  • Li, line item.
  • Img, contains information abpout images that are displayed on the webpage.
  • Src, attribute that defines the location of an image.
  • Br, creates a line of empty space, or a line break, between two breaks of contyent.
  • A, anchor, creates links to the same webpage or other webapges.
  • Href, points to the URL for the link.
  • Alt, contains a text stringthat describes the appearance and funtionality of an image.
  • Section, semantic elements that define a section in a documentthat conatains thematic content.
  • Id, a way to identify elements and select them.

CSS

  • Cascading Style Sheets, programming language used by developers to define how webpage content is presented to users.CSS defines the color, aesthetics, visual layout, and organization. We use CSS to define style rules for specific elements of an HTML page and format them using a wide range of style options.
  • Inline CSS, apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only.
  • Internal Style Sheet, contains CSS rules for the webpage in the head section of the HTML file.
  • External CSS style sheet, allows developers to keep all their CSS rukes in a separate file, which makes design changes easier.
  • Separation of concerns, separating code into different files based on use.
  • Declarations, containstwo important components; the CSS property we want to apply and the value of the property.
  • Margin, how much space we want around the outside of an element.
  • Padding, adds space around the content inside an element.
  • DRY, Do not repeat yourself, a coding principal that encourages developers to resue or share pieces of code in order to redue the number of lines of code that need to be written, or maintained.

Git

  • Git Status, checks what branch we are currently on.
  • Git Checkout -b branch-name, creates a new branch and switches to it.
  • Bash=Shell
  • pwd, print working directory
  • cd, change directory.
  • ls, shows a list of files and folders.
  • cd .., goes one step backwards in the directory.
  • mkdir, make a directory.
  • touch, create a new file.
  • start, opens file.
  • SSH Key, access credential for the secure shell network protocol.
  • MVP, minimal viable product.
  • Repo, every project on Github is called a repository, folder that contains all of the files and revision history.
  • Branch, workspace where you can make changes that dont affect the live site.
  • Markdown, the way content files are formatted in Github. A way to write for the web using plain text, easily converts to HTML. Use .md file extension.
  • Commit changes, saving a new version of a file youre working on whenever you add, edit, or delete a file.
  • Pull request, asking for changes on a branch to merge into another branch. Way to discuss and review your work with others.
  • Merge, when a PR is approved changes are "pulled in" ormerged into a main branch. Changes publish/deploy to the live site once this happens.
  • Issue, way to report new tasks or content fixes, track progress on a project board, open an issue for a new task and close once done.
  • Integrated Development Environment, type of software used by developers that combines many of the features needed by developers to do their work on a single interface, known as a graphical user interface GUI.

JavaScript

  • A Variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • JavaScript, is a programming language that adds interactivity to your website.
  • Application Programming Interfaces, API, built into we browsers, providing functionality such asdynamically creating HTML and setting CSS styles; collecting and manipulating a video stream from a users webcam, or generating 3D graphics and audio samples.
  • Third Party API, allow developers to incorporate funcionality in sites from other content providers, such, as, Twitter or Facebook.
  • Third party frameworks and libraries, can be applied to HTML to accelerate the work of building sites and applications.
  • Var, used to describe a variable.
  • =, assignment operator to assign the specific datathat we want stored in our variable.
  • Each line of JavaScript ends with a semicolon.
  • Console.log, output a message to the web consoleby creating an argument.
  • Control flow, is the order in which a computer executes code in a script. In JavaScript, code runs from the top of the page, to the bottom of the page.
  • If statement, a way of introducing desicion-making into our code. Code will only execute if the statement meets a condition, or is true.
  • Else if, used if there is more than one condition.
  • The Strict Equality Operator, ===, checks to see if two values are equal, and returns a Boolean result "true" if equal, or "false" if not equal.
  • Array, a single variable that is used to hold a group of data.
  • To work with individual data items inside an array, we need the location of the data item in the array. This location is identified with a unique number called an index. Using each piece of datas index, or location number, we can access individual data inside the array.
  • Arrays are zero-indexed and sequential. The first item in any array has an index of 0 not 1. The indices, or location identifiers, will always be whole numbers that increase by 1 each time.
  • For loop, uses the predictable pattern of indices to perfeorm a task on all the items in an array by allowing a single code block to be executed over, and over.
  • The length of an array can be found by using the built in, length, property.
  • A for loop contains three important statements;
  • The first statement determines the starting point for our loop. Scince we want to start at the beginning of the array, we declare a variable of "x" and give it a value of "0".
  • The second statement is the condition. As long as the number in the variable "x" is less then the length of the array, the loop will keep running.
  • The final statement is what allows the array to iterate over each item. Since indices follow a predictable pattern, we know the number increases by 1 each time. The code "x++" is a shortcut way of writting x+1. Now every time you code loops "x" will increase by 1.
  • Commenting out code, makes it inactive without deleting it, just in case you need it later. Put /* before and */ after the block of code.
  • OOP, object oriented programming.
  • Function, A function is different from a "for loop" or "conditional statement" because funtions do not automatically execute when the JavaScript file is run. A funtions name is followed by ().
